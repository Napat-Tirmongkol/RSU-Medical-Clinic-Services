# คู่มือการใช้งานระบบ "นักศึกษาทุน" (Scholarship)

ระบบจัดการชั่วโมงทำงานของนักศึกษาทุนที่คลินิก — รวมถึงเปิดรอบงาน, จองรอบ, เช็คอิน/เช็คเอาต์ด้วย GPS, อนุมัติชั่วโมง, รายงาน

> **Audience:** ผู้ใช้งาน 2 ฝั่ง — **Admin** (เจ้าหน้าที่คลินิก) และ **Student** (นักศึกษาทุน)

---

## 1. ภาพรวมการทำงาน

```
┌─────────────────────────┐         ┌──────────────────────────┐
│  Admin (เจ้าหน้าที่)    │         │  Student (นักศึกษาทุน)  │
├─────────────────────────┤         ├──────────────────────────┤
│ ① เปิดรอบงาน            │ ──────► │ ② เลือกรอบ + จอง         │
│   (date × time × cap)   │         │   (ผ่านปฏิทินใน LINE)    │
│                         │         │                          │
│ ④ อนุมัติเข้า/ออกงาน   │ ◄────── │ ③ Clock-in / Clock-out    │
│   (ถ้าเปิด require)     │         │   (GPS + เลือกประเภท)    │
│                         │         │                          │
│ ⑤ ดูรายงาน + Export    │         │ ⑥ ดูประวัติของตัวเอง     │
└─────────────────────────┘         └──────────────────────────┘
```

**ประเภทค่าตอบแทน 2 แบบ** — นักศึกษาเลือกเองตอน **clock-out**:
- 🎓 **ส่งชั่วโมงทุน** — นับเป็นชั่วโมงสะสมตามเกณฑ์ทุน
- 💰 **ค่าตอบแทน** — รับเป็นเงิน ตามอัตรา ฿/ชม. ที่ admin ตั้ง

---

## 2. คู่มือสำหรับ Admin

### 2.1 การเข้าใช้งาน
- เปิด **Portal** → sidebar เมนู **"นักศึกษาทุน"** (อยู่ในกลุ่ม "ข้อมูลหลัก")
- ต้องมีสิทธิ์ `access_scholarship` หรือ role `superadmin` (ตั้งค่าใน Identity & Governance)

### 2.2 แท็บ Dashboard
- **KPI cards:** จำนวนนักศึกษา active, รออนุมัติ, รอบวันนี้, ชั่วโมงรวม (ทุน + paid + เงิน)
- **กราฟ:** ชั่วโมงรายวัน 30 วันล่าสุด + สัดส่วน hours vs paid

### 2.3 แท็บ "นักศึกษาทุน" — จัดการบัญชี
- **เพิ่มนักศึกษา:** กรอก
  - รหัสนักศึกษา · คณะ · ประเภททุน · ภาคเรียน
  - **max_hours** = เพดานชั่วโมงรวม (เกินแล้วระบบยังให้ทำ แต่ admin ใช้ตรวจสอบ)
  - **สถานะ** active/inactive (inactive ห้ามเช็คอิน)

### 2.4 แท็บ "เปิดรอบงาน" — สร้าง slot ให้นักศึกษาจอง

**ปุ่ม "+ เปิดรอบใหม่"** เปิด modal:
| ฟิลด์ | คำอธิบาย |
|---|---|
| วันที่เริ่ม / สิ้นสุด | ช่วงวันที่จะเปิดรอบ |
| **Quick presets** | "วันนี้ / +1 สัปดาห์ / +1 เดือน / +ภาคเรียน (4 เดือน)" — คลิกเดียวตั้งช่วง |
| วันในสัปดาห์ | เลือก DoW ที่จะเปิด (ไม่เปิดเสาร์-อาทิตย์ก็ได้) |
| รอบเวลาในแต่ละวัน | กดเพิ่มได้หลายช่วง เช่น 08:00–12:00 + 13:00–17:00 |
| รับนักศึกษา (คน/รอบ) | จำนวนสูงสุดต่อ slot |
| หมายเหตุ | เช่น "ห้องตรวจ A" |

> 💡 **ไม่ต้องเลือกประเภท (ทุน/เงิน) ที่นี่** — นักศึกษาจะเลือกเองตอน clock-out

ระบบจะสร้าง slot = (จำนวนวันที่ตรง DoW) × (จำนวนรอบเวลา)
เช่น 1 เดือน × จันทร์-ศุกร์ × 2 รอบ/วัน = ~40 slots

### 2.5 แท็บ "ปฏิทิน" — มุมมองภาพรวมรายเดือน

ตารางเดือน 7×N เซลล์ · ปุ่ม **← / วันนี้ / →**

| สีเซลล์ | ความหมาย |
|---|---|
| 🟢 เขียว | มีรอบเปิด ยังว่าง |
| 🟡 เหลือง | เต็มทุกรอบ |
| 🔴 แดงจาง + badge "หยุด" | คลินิกหยุด (จาก `sys_clinic_hours`) |
| ⬜ ขาว | ไม่มีรอบ |

**การโต้ตอบ:**
- **คลิกเซลล์ว่างในอนาคต** → modal เปิดรอบใหม่ขึ้นทันที + วันที่ pre-fill ✨
- **คลิกเซลล์มี slot** → modal โชว์รายละเอียดวัน:
  - subtitle: "X รอบ · Y จอง · Z เช็คอินแล้ว" (เห็นเช็คอินจริงทันที)
  - per slot: เวลา + จำนวนจอง/สูงสุด + badge `Z✓/Y` (เช็คอินแล้ว/จอง)
  - per student: ✓ chip ฟ้าถ้าเช็คอินแล้ว, chip เขียวถ้ายังไม่มา
  - ปุ่ม **"แก้ไข" / "ลบ"** ต่อ slot
  - ปุ่ม **"+ เพิ่มรอบในวันนี้"** ที่หัว modal

### 2.6 แท็บ "รออนุมัติ" — อนุมัติ clock log

แสดงเฉพาะ log ที่ `status = 'pending'`:
- กด **"อนุมัติ"** → log กลายเป็น `approved` (ชั่วโมงนับเข้าระบบ)
- กด **"ปฏิเสธ"** → ใส่เหตุผล → `rejected` (ไม่นับ + student เห็นเหตุผล)

> **Tip:** ปิด `require_approval` ในตั้งค่า → log จะถูก approve อัตโนมัติทุกครั้ง (admin ไม่ต้องเช็ค manual)

### 2.7 แท็บ "ตารางกะ" — Admin assign shift ตรงๆ

ใช้กรณีที่ admin ต้องการ "บังคับ" ตารางให้นักศึกษาคนใดคนหนึ่ง (ไม่ผ่านระบบจอง slot):
- เลือกนักศึกษา + วันที่ + เวลา + ประเภท (ที่นี่ยังเลือกได้ — แต่ student override ที่ clock-out ได้)

### 2.8 แท็บ "รายงาน"

- เลือกช่วงวันที่
- **Export CSV** — โหลดเป็น Excel ได้
- คอลัมน์: นักศึกษา, รหัส, ชั่วโมงทุน, ค่าตอบแทน (ชม.), เงิน

### 2.9 แท็บ "ตั้งค่า"

| ฟิลด์ | Default | คำอธิบาย |
|---|---|---|
| ตรวจ GPS | เปิด | บังคับให้ user อยู่ในรัศมีคลินิก |
| พิกัดคลินิก | — | Lat, Lng (กดปุ่ม "ใช้ตำแหน่งปัจจุบัน" ได้) |
| รัศมี (เมตร) | 50 | นอกรัศมีนี้ → reject |
| Grace before | 15 นาที | เข้างานก่อนกะเริ่ม X นาทีได้ |
| ต้องอนุมัติ | เปิด | log = pending รอ admin (ปิด = approve ทันที) |
| อัตรา ฿/ชม. | 0 | สำหรับคำนวณค่าตอบแทน |
| ยกเลิกรอบก่อน N ชม. | 24 | cutoff student ยกเลิกการจอง |

## 3. คู่มือสำหรับนักศึกษาทุน

### 3.1 การเข้าใช้งาน
- เปิดผ่าน **LINE** → เมนู "เก็บชั่วโมงทุน" หรือ scan QR
- ต้องเป็นนักศึกษาทุน status `active` (admin เพิ่มให้)

### 3.2 หน้าแรก — โครงสร้าง

```
┌──────────────────────────────┐
│  สวัสดี [ชื่อ]                │ ← Profile
│  เวลาปัจจุบัน 14:30:45         │
├──────────────────────────────┤
│  กะวันนี้: 13:00–17:00 (ทุน)  │ ← วันที่มีรอบ (ถ้าจองไว้)
│                              │
│  [ ▶ เริ่มงาน ]              │ ← ปุ่ม clock_in/clock_out
├──────────────────────────────┤
│ ส่งชม.ทุน 12   ค่าตอบแทน 8    │ ← KPI เดือนนี้
│ เงิน 1,600 ฿                  │
├──────────────────────────────┤
│  📅 ปฏิทินรอบงาน              │ ← เลือกวันจอง (ดูข้อ 3.3)
├──────────────────────────────┤
│  🔖 รอบที่จองไว้              │ ← ดู + กดยกเลิกได้
├──────────────────────────────┤
│  ⏱ ประวัติล่าสุด              │
└──────────────────────────────┘
```

### 3.3 จองรอบงาน

1. **ดูปฏิทิน** — สีบอกสถานะ:
   - 🟢 เขียว = มีรอบเปิด
   - 🔵 ฟ้า + "จองแล้ว" = คุณจองไว้แล้ว
   - 🟡 เหลือง = เต็มแล้ว
   - 🔴 แดง + "หยุด" = คลินิกหยุด
2. **แตะวัน** → modal โชว์รอบในวันนั้น
3. **กด "จอง"** ที่รอบที่ต้องการ → ยืนยัน → จองสำเร็จ
4. รอบจะปรากฏใน section **"รอบที่จองไว้"**

### 3.4 ยกเลิกการจอง

- ไป section **"รอบที่จองไว้"** → กดปุ่ม **✕** ที่รอบ
- ใส่เหตุผล (ไม่บังคับ) → ยืนยัน
- **ยกเลิกได้ก่อนรอบเริ่ม ≥ 24 ชม.** (admin ตั้งค่า default; เลย cutoff แล้วยกเลิกไม่ได้)

### 3.5 เข้างาน (Clock-in)

1. กดปุ่มเขียว **"▶ เริ่มงาน"** หรือ **"เข้างาน"**
2. ระบบขอ **GPS** → อนุญาตในเบราว์เซอร์/LINE
3. ตรวจระยะ — ต้องอยู่ในรัศมี (เช่น 50 ม.) จากคลินิก
4. ส่งคำขอ — ขึ้น "เข้างานสำเร็จ" (หรือ "ส่งคำขอ — รอเจ้าหน้าที่อนุมัติ" ถ้า require_approval)

> ⚠️ **ถ้า GPS reject** เพราะอยู่นอกรัศมี → ติดต่อ admin ให้ตรวจสอบ/ปิด GPS check

### 3.6 ออกงาน (Clock-out) — **เลือกประเภทตอนนี้**

1. กดปุ่มแดง **"⏹ ออกงาน"**
2. **Modal เลือกประเภท** ขึ้นมา:
   - 🎓 **ส่งชั่วโมงทุน** — เก็บเป็นชั่วโมงตามเกณฑ์ทุน
   - 💰 **ค่าตอบแทน** — รับเงินตามอัตรา
3. **Modal สรุปยืนยัน** โชว์:
   - เข้างานเมื่อ HH:MM
   - ประเภทที่เลือก
   - ทำงานจริง X ชม. Y นาที
   - **นับเข้าระบบ Z ชม.** (ปัดลงเป็นชั่วโมงเต็ม — 2 ชม. 59 นาที = 2 ชม.)
4. ยืนยัน → GPS → ส่งคำขอ

### 3.7 ดูประวัติ

- หน้าหลัก: section **"ประวัติล่าสุด"** แสดง 5 รายการ
- กด **"ดูทั้งหมด"** → ไปหน้า `scholarship_history.php`

## 4. FAQ / Troubleshooting

| ปัญหา | สาเหตุ / วิธีแก้ |
|---|---|
| Student: "ยังไม่มีรอบว่าง" แต่ admin บอกเปิดแล้ว | กดปุ่ม **🔄 โหลดอีกครั้ง** หรือ refresh หน้า · ตรวจให้ slot status = `open` |
| กด clock-out ขึ้น "ยังไม่ได้เข้างาน" | log clock-in ก่อนหน้าถูก admin ปฏิเสธ → ต้อง clock-in ใหม่ |
| Student อยู่นอกพื้นที่ GPS | admin ปิด `GPS_REQUIRED` ในตั้งค่าได้ (manual approve แทน) |
| Admin: ลบ slot แล้วยังเห็น "ยกเลิก" ในตาราง | เป็น filter "ทุกสถานะ" — เปลี่ยนเป็น filter "เปิดรับ" จะหาย (หรือกดลบใน slot ที่ "ยกเลิก" จะลบจริง) |
| ชั่วโมงนับผิดประเภท | นักศึกษากด clock-out → เลือกประเภทผิด · admin แก้โดยใช้ "manual adjustments" tab |
| GPS เพี้ยน — ผ่านรัศมีทั้งที่ไม่ได้อยู่คลินิก | ปกติของอุปกรณ์ในร่ม/ตึกใหญ่ · admin สามารถ reject log นั้นๆ ได้ใน "รออนุมัติ" |

## 5. โครงสร้างฐานข้อมูล (อ้างอิงเร็ว)

| ตาราง | บทบาท |
|---|---|
| `sys_scholarship_students` | ข้อมูลนักศึกษาทุน (1:1 กับ sys_users) |
| `sys_scholarship_slots` | รอบที่ admin เปิดให้จอง |
| `sys_scholarship_slot_bookings` | การจองของนักศึกษา (UNIQUE slot_id + student_id) |
| `sys_scholarship_shifts` | กะ — สร้างอัตโนมัติเมื่อจอง slot (`slot_id` ผูกกลับ) |
| `sys_scholarship_clock_logs` | log clock-in / clock-out + GPS + status |
| `sys_scholarship_manual_adjustments` | admin ปรับชั่วโมงด้วยมือ (audit trail แยก) |
| `sys_scholarship_settings` | ตั้งค่าระบบ (1 row) |
