# 🏥 RSU Medical Hub
### บริการสุขภาพ มหาวิทยาลัยรังสิต — ในมือเดียว

- ✨ Single sign-on ผ่าน LINE
- 🛡️ PDPA-compliant ตามมาตรฐานล่าสุด
- 📱 Mobile-first · ใช้งานได้ทันทีโดยไม่ต้องโหลดแอป

> สวัสดีครับ วันนี้ผมจะพาทุกท่านชม RSU Medical Hub — แพลตฟอร์มสุขภาพดิจิทัลของศูนย์บริการสุขภาพ มหาวิทยาลัยรังสิต ที่รวมทุกบริการของคลินิกเข้าไว้ในที่เดียว เข้าผ่าน LINE ได้ในไม่กี่วินาที โดยไม่ต้องโหลดแอปเพิ่ม

# 🤔 ก่อนหน้านี้

เข้าถึงบริการสุขภาพในมหาวิทยาลัย:

- 🏃 เดินไปคลินิกเพื่อตรวจสอบสิทธิ์ประกัน
- 📞 โทรจองคิวล่วงหน้า
- 📄 กรอกใบสมัคร / เอกสารยืนยันตัวตน
- 💉 ประวัติวัคซีนกระจัดกระจาย — ไม่มีที่เดียวให้ดู

> ก่อนหน้านี้ การเข้าถึงบริการสุขภาพของม.รังสิต ต้องผ่านขั้นตอนหลายอย่าง — เดินไปคลินิกเพื่อตรวจสอบสิทธิ์ประกัน โทรจองคิวล่วงหน้า กรอกใบสมัครเป็นกระดาษ ประวัติวัคซีนต้องขอเอาเอง

# ⚡ Login ใน 10 วินาที

1. แตะปุ่ม **"เข้าสู่ระบบด้วย LINE"**
2. ทบทวนนโยบาย PDPA (มาตรา 24 + 26)
3. เข้าใช้งานทันที — ไม่ต้องจำรหัสผ่าน

![LINE login → PDPA consent → hub](placeholder:login-flow.png)

> เริ่มจากการ login — แค่แตะปุ่ม LINE ผู้ใช้จะเข้าระบบได้ พร้อมทบทวนนโยบายความเป็นส่วนตัวตาม พ.ร.บ.คุ้มครองข้อมูลส่วนบุคคล โดยแยกความยินยอมเป็นข้อมูลทั่วไป มาตรา 24 และข้อมูลอ่อนไหว มาตรา 26 อย่างชัดเจน ตามมาตรฐานล่าสุด

# 👤 Profile Hub ส่วนตัว

![Profile card with green Verified badge](placeholder:profile-hub.png)

- 🪪 Member ID + เลขประจำตัวสำหรับ QR check-in
- ✅ ป้าย **Verified** เมื่อข้อมูลตัวตนครบ (Mod-11 + DOB + เบอร์)
- 🎓 คณะ / สังกัด · ประเภทผู้ใช้
- 🛡️ สถานะประกันอุบัติเหตุ + ยอดคุ้มครอง

> หน้าหลักของผู้ใช้แสดงข้อมูลส่วนตัวพร้อมป้าย Verified ซึ่งจะขึ้นเมื่อระบบยืนยันได้ว่าข้อมูลตัวตนครบ — เลขบัตรประชาชนถูกต้องตามอัลกอริทึม วันเดือนปีเกิด และเบอร์โทร ผู้ใช้เห็นสถานะประกันอุบัติเหตุของตัวเองได้ทันที พร้อมยอดคุ้มครองและวันหมดอายุ

# 💉 ประวัติวัคซีนของฉัน

![Vaccination card showing dose history + next due](placeholder:vaccine-history.png)

- 📋 ทุก dose ที่เคยฉีด · lot/ผู้ผลิต/เจ้าหน้าที่

# 🛡️ Privacy by Design

ทุก action บนแอปอยู่ภายใต้:

- ✅ PDPA v2 — granular consent มาตรา 24 + มาตรา 26
- ✅ บันทึกหลักฐาน (IP + timestamp + version hash)
- ✅ Audit log ทุกการเข้าถึงข้อมูลอ่อนไหว
- ✅ สิทธิ์ถอนความยินยอม / ขอข้อมูล (DSAR) ตามมาตรา 30

> ทุกขั้นตอนถูกออกแบบให้ปลอดภัยตามมาตรฐาน PDPA — ความยินยอมแยกประเภทตามมาตราของ พ.ร.บ. บันทึกหลักฐานพร้อม IP และเวลา audit log ทุกครั้งที่เข้าถึงข้อมูลอ่อนไหว ผู้ใช้สามารถถอนความยินยอมหรือร้องขอข้อมูลของตัวเองได้ตามสิทธิ์มาตรา 30
